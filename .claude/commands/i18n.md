# /i18n

Shortcut to invoke the `add-translation` skill with optional arguments.

## If arguments are provided

Parse `$ARGUMENTS` as `<namespace.key> "<English source>"`. Add the key to all four locale files (`en`, `es`, `fr`, `ar`) following the rules in the `add-translation` skill: reuse existing namespaces, respect the proper-noun list, use the prefix/linkText/suffix pattern for sentences with inline links, and flag any hard-coded English in nearby components that should be migrated at the same time.

## If no arguments are provided

Ask the user for:

1. The component file where the string will be used
2. The English source copy
3. Whether it's a standalone string, a button label, a sentence with an inline link, or chip/wizard copy

Then proceed as above.

## Always report

```markdown
## Added
- `namespace.key` — used in `<file>:<line>`

## Locale coverage
- [x] en: "..."
- [x] es: "..."
- [x] fr: "..."
- [x] ar: "..."

## Follow-ups
- (hard-coded English siblings to migrate, tests needing a language reset)
```
